# TodoListApp 도메인 정의서

**버전:** 1.0
**작성일:** 2026-05-12
**작성자:** Business Analyst

---

## 1. 프로젝트 개요

| 항목      | 내용                                                                                               |
| --------- | -------------------------------------------------------------------------------------------------- |
| 앱 이름   | TodoListApp                                                                                        |
| 목적      | 인증된 사용자가 카테고리 기반으로 할일을 체계적으로 관리할 수 있는 개인화된 할일 관리 애플리케이션 |
| 핵심 문제 | 사용자 인증 기반의 개인 데이터 격리, 카테고리별 할일 분류 관리 부재                                |

---

## 2. 핵심 도메인 용어 (Ubiquitous Language)

| 용어                                       | 정의                                                              |
| ------------------------------------------ | ----------------------------------------------------------------- |
| **사용자 (User)**                          | 회원가입 후 인증을 통해 앱을 사용하는 주체                        |
| **할일 (Todo)**                            | 사용자가 등록하고 관리하는 작업 단위                              |
| **카테고리 (Category)**                    | 할일을 분류하는 그룹. 기본 카테고리와 사용자 정의 카테고리로 구분 |
| **기본 카테고리 (Default Category)**       | 시스템이 모든 사용자에게 제공하는 사전 정의된 카테고리            |
| **사용자 정의 카테고리 (Custom Category)** | 사용자가 직접 생성한 카테고리                                     |
| **완료 여부 (Completion Status)**          | 할일의 처리 상태 (미완료 / 완료)                                  |
| **일정 종료 여부 (Schedule Status)**       | 종료예정일 기준으로 판단하는 일정 초과 여부 (진행중 / 기간초과)   |
| **회원가입 (Registration)**                | 이메일, 비밀번호 등 기본 정보를 등록하여 계정을 생성하는 행위     |
| **인증 (Authentication)**                  | 등록된 자격증명으로 신원을 확인하고 접근 권한을 부여하는 절차     |

## 3. 도메인 모델

### 3.1 User (사용자)

| 속성       | 타입     | 설명                    | 제약             |
| ---------- | -------- | ----------------------- | ---------------- |
| id         | UUID     | 사용자 고유 식별자      | PK, 자동생성     |
| email      | String   | 이메일 주소 (로그인 ID) | Unique, Not Null |
| password   | String   | 암호화된 비밀번호       | Not Null         |
| name       | String   | 사용자 이름             | Not Null         |
| created_at | DateTime | 가입일시                | 자동생성         |
| updated_at | DateTime | 최종 수정일시           | 자동갱신         |

### 3.2 Category (카테고리)

| 속성       | 타입     | 설명                               | 제약                |
| ---------- | -------- | ---------------------------------- | ------------------- |
| id         | UUID     | 카테고리 고유 식별자               | PK, 자동생성        |
| name       | String   | 카테고리 이름                      | Not Null            |
| is_default | Boolean  | 기본 카테고리 여부                 | Default: false      |
| user_id    | UUID     | 소유 사용자 (기본 카테고리는 NULL) | FK → User, Nullable |
| created_at | DateTime | 생성일시                           | 자동생성            |

### 3.3 Todo (할일)

| 속성         | 타입     | 설명             | 제약                    |
| ------------ | -------- | ---------------- | ----------------------- |
| id           | UUID     | 할일 고유 식별자 | PK, 자동생성            |
| user_id      | UUID     | 소유 사용자      | FK → User, Not Null     |
| category_id  | UUID     | 분류 카테고리    | FK → Category, Not Null |
| title        | String   | 할일 제목        | Not Null                |
| description  | String   | 할일 설명        | Nullable                |
| start_date   | Date     | 시작일           | Not Null                |
| due_date     | Date     | 종료예정일       | Not Null                |
| is_completed | Boolean  | 완료 여부        | Default: false          |
| created_at   | DateTime | 등록일시         | 자동생성                |
| updated_at   | DateTime | 최종 수정일시    | 자동갱신                |

### 3.4 엔티티 관계

```
User (1) ──── (N) Todo
User (1) ──── (N) Category
Category (1) ──── (N) Todo
```

## 4. 비즈니스 규칙

### 인증 규칙

- BR-01: 미인증 사용자는 할일 및 카테고리 관련 기능에 접근할 수 없다.
- BR-02: 이메일은 시스템 전체에서 유일해야 하며, 로그인 ID로 사용된다.

### 데이터 격리 규칙

- BR-03: 사용자는 자신이 등록한 할일만 조회, 수정, 삭제할 수 있다.
- BR-04: 사용자 정의 카테고리는 해당 카테고리를 생성한 사용자만 사용할 수 있다.

### 카테고리 규칙

- BR-05: 기본 카테고리는 시스템이 관리하며 모든 사용자가 공통으로 사용한다.
- BR-06: 사용자는 기본 카테고리 외에 사용자 정의 카테고리를 추가할 수 있다.
- BR-07: 할일 등록 시 반드시 하나의 카테고리를 지정해야 한다.

### 할일 규칙

- BR-08: 종료예정일은 시작일보다 이전일 수 없다.
- BR-09: 완료 처리된 할일은 완료 취소(미완료 복원)가 가능하다.

## 5. 주요 유스케이스

| ID    | 유스케이스     | 액터          | 설명                                                               |
| ----- | -------------- | ------------- | ------------------------------------------------------------------ |
| UC-01 | 회원가입       | 비인증 사용자 | 이메일, 비밀번호, 이름을 입력하여 계정 생성                        |
| UC-02 | 로그인         | 비인증 사용자 | 이메일과 비밀번호로 인증 후 서비스 접근                            |
| UC-03 | 개인정보 수정  | 인증된 사용자 | 이름 등 개인정보를 수정                                            |
| UC-04 | 할일 등록      | 인증된 사용자 | 제목, 설명, 시작일, 종료예정일, 카테고리를 입력하여 할일 생성      |
| UC-05 | 할일 수정      | 인증된 사용자 | 등록된 할일의 내용을 수정                                          |
| UC-06 | 할일 삭제      | 인증된 사용자 | 등록된 할일을 삭제                                                 |
| UC-07 | 할일 완료 처리 | 인증된 사용자 | 할일의 완료 여부를 토글 (완료 ↔ 미완료)                            |
| UC-08 | 할일 목록 조회 | 인증된 사용자 | 카테고리, 일정 종료 여부, 완료 여부 필터를 조합하여 할일 목록 조회 |
| UC-09 | 카테고리 추가  | 인증된 사용자 | 사용자 정의 카테고리 생성                                          |
| UC-10 | 카테고리 삭제  | 인증된 사용자 | 사용자 정의 카테고리 삭제                                          |

## 6. 도메인 이벤트

| 이벤트             | 트리거 조건               | 후속 처리                      |
| ------------------ | ------------------------- | ------------------------------ |
| UserRegistered     | 회원가입 완료             | 기본 카테고리 자동 할당        |
| UserLoggedIn       | 로그인 성공               | 인증 토큰 발급                 |
| UserProfileUpdated | 개인정보 수정 완료        | 변경 이력 기록                 |
| TodoCreated        | 할일 등록 완료            | 할일 목록 갱신                 |
| TodoUpdated        | 할일 수정 완료            | 할일 목록 갱신                 |
| TodoDeleted        | 할일 삭제 완료            | 할일 목록 갱신                 |
| TodoCompleted      | 완료 상태로 변경          | 완료 처리 일시 기록            |
| TodoReopened       | 미완료 상태로 복원        | 완료 처리 일시 초기화          |
| CategoryCreated    | 사용자 정의 카테고리 생성 | 카테고리 목록 갱신             |
| CategoryDeleted    | 사용자 정의 카테고리 삭제 | 연관 할일의 카테고리 처리 필요 |
