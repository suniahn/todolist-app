### SCN-09: Access Token 만료 후 자동 재발급 흐름 ⚠️ PARTIAL

**관찰된 동작:**
- Zustand authStore는 메모리 기반으로, 페이지 새로고침(browser_navigate) 시 토큰이 소실됨
- 소실 시 보호 경로(`/todos/new`)에서 `/login`으로 즉시 리다이렉트 확인 → 미인증 차단 정상 동작

**미확인 항목:**
- 1시간 뒤 실제 Access Token 만료 후 Refresh Token으로 자동 재발급 시나리오 (시간 조작 불가)
- Axios 인터셉터가 401 감지 후 `/auth/refresh` 호출 여부

**권장 대안:**
- 단위 테스트에서 `Date.now()` 모킹 또는 JWT 만료 시간 단축 후 검증

## 발견된 이슈

### ISSUE-01: 직접 URL 탐색 시 인증 상태 초기화 (정상 동작)
- **현상:** `browser_navigate`로 보호 경로 직접 이동 시 `/login` 리다이렉트
- **원인:** Zustand authStore가 메모리 기반 (페이지 새로고침 시 토큰 소실)
- **평가:** SCN-09 사후 조건의 설계 의도와 일치. 이슈 아님.
- **관련 시나리오:** SCN-09 사후조건 - "페이지 새로고침 시 초기화"

### ISSUE-02: 스크린샷 저장 미실행
- **현상:** 테스트 중 스크린샷을 별도 저장하지 않음
- **원인:** playwright-mcp 직접 조작 방식에서 자동 스크린샷 미설정
- **권장:** 실패 케이스 재현 시 `browser_take_screenshot` 활용

## 환경 특이사항

- **토큰 저장 방식:** Zustand 메모리 기반 → 탭/새로고침 시 재로그인 필요 (설계 의도)
- **SPA 라우팅:** `browser_navigate`로 직접 접근 시 인증 상태 소실, 내부 링크/버튼 클릭으로 탐색 필요
- **기본 카테고리:** `is_default=true`인 4종(업무, 개인, 건강, 쇼핑)이 모든 사용자에게 공유
